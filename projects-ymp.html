Services:


1. Agent : This module consists of the configurable API call function. This function can be called from wherever this module is imported by passing the parameters like request name(ex. post, get,etc ), endpoint , body, and a Boolean for the loader display. This function then returns the response body asynchronously after some conventional error handling.
2. Node server : This server sends build file of the server side rendered project as a response if the request is made from browser. If the request is made from crawler bots , the server redirects those requests to the project which is server side rendered so that they can access meta information in purposes of SEO.
3. SSR project : This is a server side rendered project running on a port that the node server directs crawler bot requests to. This project only throws the meta information to those bots for SEO purposes as they can’t be accessed in server side rendered (SPA) project.

Product CRUD with image crop and resize UI

Actions from user :
1. Select an image file from the local memory storage. (Or click a picture from the camera if on mobile phone.)
2. Crop the image as required.
3. Fill up the form and press upload.


Processings done on image :
1. Validation checks on the cropped image for minimum height and width as per business logic. Rejection of the respective image if not matched the criteria.
2. In case the cropped image is larger than the maximum limits, resize it to maximum limits maintaining the aspect ratio.
3. Multiple image upload is possible. the code structure allows to change the maximum upload limit by changing the value of variable max_upload.
4. The artwork profile is created using only the first image along with the form data which returns the artwork id and the other images are attached to that artwork id to avoid the risks of losing any data.
